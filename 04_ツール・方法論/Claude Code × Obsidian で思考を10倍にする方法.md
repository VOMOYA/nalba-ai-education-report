# Claude Code × Obsidian で思考を10倍にする方法

## 概要

Claude Code と Obsidian を組み合わせることで、AIを「推論マシン」として操り、自分の思考を飛躍的に高度化できるという内容。田原真人氏による実演を交えた解説。

---

## 1. 学習の2つの段階

### 覚えて再現する（機械が得意）
- 外部から入力 → 決められたパターンでアウトプット
- コンピューターや機械に代替されてきた領域

### 理解して応用する（かつては人間だけ）
- 外部情報 ＋ 記憶 を組み合わせて**推論**する
- →  AIが「推論」できるようになったことで、この領域にもAIが参入

---

## 2. AI活用の第1段階：言語・理解の壁を壊す

- どんな言語の本も翻訳して読み込める
- 難しい内容をAIが自分に合わせてチュータリングしてくれる
- → 情報のインプット入口が大きく広がった

---

## 3. AIを「推論マシン」として使う

> AIは情報をグツグツ煮込んで料理にするお鍋のようなもの

### 推論の調理方法（情報の扱い方）
- **帰納法**：複数の事例から共通点を導く
- **演繹法**：一般論から個別の結論を導く
- **アブダクション**：最もありそうな説明を推定する

### 実例：NPOの社会インパクト測定
1. NPOの活動内容
2. 社会的インパクトの測定手法
3. 過去の先行事例

→ これらを全部AIに入れて「スコアリング手法に基づいて評価し、過去事例と比較して」と指示する

### ペルソナ活用の例
```
あなたは川喜田二郎（KJ法の創始者）です。
以下の付箋をKJ法でまとめてアウトプットしてください。
```
→ 自分でKJ法をやるのではなく、**AIにメソッドを覚え込ませてやらせる**

---

## 4. 情報が溢れる問題

- ChatGPT / Gemini との対話で毎日大量の情報が生成される
- 手作業でフォルダ管理 → 量が増えるとどこに何があるか分からなくなる
- ファイルをAIに手で貼り付けて渡す作業が発生 → **思考が止まる**

---

## 5. Obsidian とは

### 起源：ルーマンのZettelkasten（ツェッテルカステン）
- ニクラス・ルーマン（一般システム論の確立者）
- 他の研究者の**10倍**の論文を執筆
- 方法：カードに情報をまとめ、タグで関連付け → 論文執筆時にタグでカードを集めて並べる

### Obsidianの仕組み
- カード（ノート）= Markdownファイル
- タグ = ファイル間のリンク
- グラフビュー = ノート同士の関係をネットワーク図で可視化
- **パソコン上の普通のフォルダに保存されるシンプルな構造**（NotionのようにアプリにロックインされないN

### 限界
- 100個程度のファイルなら管理できる
- それを超えるとタグの貼り替えや新規ファイルの整理が破綻する

---

## 6. Claude Code × Obsidian の組み合わせ

### Claude Code とは
- プログラミング用AIツール（バイブコーディング）
- パソコンにインストールし、クラウドのAIとやり取りしながら**ファイルを直接読み書き**できる
- 例：「テトリスを作って」→ テトリスのファイルをローカルに生成

### 組み合わせの核心

```
Obsidianのフォルダ（Markdownファイル群）
        ↕ 直接アクセス
    Claude Code
        ↕
      AIで推論
```

1. Claude Codeが**Obsidianのノートを直接読み込む**
2. 複数ファイルを横断して推論・分析
3. アウトプットを**適切なタグ・リンク付きでObsidianに保存**
4. 新しいノートがネットワーク上の適切な位置に自動的に繋がる

### 実演内容
- 「田原の個人ブランディングサイトのカラーパターンを検討して」
  → Claude CodeがObsidianの個人ブランディング関連ファイルを自動で読み込んで回答
  → 結果を `受信箱/個人サイトのカラーパターン検討.md` として保存
- ノートパソコン2台・各3ウィンドウ = **6つの作業を並列実行**

---

## 7. なぜ Obsidian ＋ Claude Code なのか

| ツール | 理由 |
|--------|------|
| **Obsidian** | ファイルが普通のMarkdownとしてフォルダに保存されており、外部ツールから直接アクセス可能 |
| **Claude Code** | ローカルのファイルを直接読み書きできる唯一のAIツール |
| **Markdown形式** | AIとの相性が抜群。構造化・リンク・タグが自然言語と共存できる |

> Notionはファイルがアプリ内に閉じているため外部ツールからアクセスできない

---

## 8. 本質と将来性

- 本質は「**Markdownファイルのフォルダ構造をAIが全体として読んで推論する**」こと
- Claude Codeが他のツールに置き換わっても、このMarkdownフォルダ構造は残る
- Obsidianは「可視化ツール」であり、代替可能

### 思考の高度化のイメージ
```
以前：3つのファイルを手で選んでAIに貼り付けて考える
  ↓
現在：ネットワーク構造全体をClaude Codeが読み込んで推論
     → 複雑な要素を関係付けながら考えられる
     → 思考が10倍（あるいはそれ以上）に
```
