## 解説的学習論（09:17〜09:23）

### 反射 vs 解説（カレン・バラードより）

```mermaid
graph LR
    subgraph 反射["【反射 Reflection】鏡のメタファー"]
        Q1[問い] --> S1[生徒]
        S1 --> A1[一つの答え]
    end
    subgraph 解説["【解説 Diffraction】回析格子のメタファー"]
        Q2[問い] --> SA[生徒A]
        Q2 --> SB[生徒B]
        Q2 --> SC[生徒C]
        SA --> INT[干渉縞✨]
        SB --> INT
        SC --> INT
    end
```

> 生徒一人ひとりが「スリット（溝）」になる。問いが生徒を通して多様な回答に広がり、波同士が重なり合って干渉縞ができる。

### 干渉縞の2つのパターン

```mermaid
graph TD
    A[問いを投げる] --> B[多様な答えが生まれる]
    B --> C{干渉縞}
    C -->|強め合う| D["明線（建設的干渉）<br/>異なる視点が補完し合い<br/>一人では到達できない洞察が生まれる"]
    C -->|弱め合う| E["暗線（相殺的干渉）<br/>矛盾する意見がぶつかり<br/>前提が問い直される"]
    D --> H[コヒーレント：全体の調和・深い納得]
    E --> H
    H --> A
```

**例：数学の背理法＝弱め合いの典型**
```
仮定「√2 は有理数」→ どんどん進める → 矛盾！
→「前提がおかしかった」と気づく → 「無理数」という新概念に到達
```

### AIをスクリーンとして活用する

```
教師           生徒（スリット）      AI（スクリーン）
[問いを投げる] → [生徒A] ─────────────┐
               → [生徒B] ─────────────┼→ [干渉縞を可視化]
               → [生徒C] ─────────────┤
               → [生徒D] ─────────────┘
```

| 評価軸 | 内容 |
|--------|------|
| 建設的干渉（明線） | 補完し合う意見のクラスタ |
| 相殺的干渉（暗線） | 矛盾・対立する意見 |
| 分布と境界 | まだ観測されていない視点の可視化 |

## Gem作成ライブデモ（09:30〜09:43）

### 田原さんのプロセス（30分で完成）

```mermaid
flowchart TD
    A["① まずAIと雑談・文脈共有\n「カレン・バラードの解説概念について教えて」\nAI：反射から解説へ、波の重なり合いで知識が生成される…\n→ ちゃんと文脈が共有できてるかを確認"]
    --> B["② 自分のアイデアを投げかける\n「学習者集団に問いを投げると様々な回答が生成される。\n解説・回析格子と対応付けたいがどうだろう？」\nAI：非常に鋭い着眼点です…"]
    --> C["③ 設計を依頼する\n「教師が問い→生徒がスリット→AIがスクリーン。\nどんな設計にすればいい？」\nAI：入力=教師の問い＋全生徒の回答、\n評価軸3つ（明線・暗線・分布）…"]
    --> D["④ プロンプト生成を依頼\n十分に文脈が共有できたタイミングで\n「Gemのプロンプトを作成して」"]
    --> E["⑤ 細かい調整\nAI：「最初にコンテキスト・問い・回答群を入れるといい」\n田原：「それもプロンプトに入れて」\n→ 3ステップ入力をGemに組み込む"]
    --> F["✅ Gem完成（約30分）\nシステムプロンプトをコピー→Gemの指示文に貼り付け→保存"]
```

> **核心：** いきなり「作って」ではなく、まずAIと文脈を共有する。「共通了解できたよね」の状態からシステムプロンプトを作ると、「そうそうそれ！」が出来上がる。

## KJ法Gem のデモと干渉縞との比較（10:40〜10:46）

**プロンプト：**「あなたは川喜田二郎です。KJ法のプロセスをステップバイステップでやってください。」

→ 同じ参加者の回答データを入力すると…

```mermaid
graph TD
    subgraph KJ["KJ法の結果（3つの島）"]
        A["島A：対話による自己変容<br/>自分の中に眠る思考を掘り起こし枠を広げる<br/>言語化・バイアスからの解放・時間の忘却"]
        B["島B：主体性と依存の境界線<br/>考える主体を維持できるか<br/>AIの言いなりにならないマインドセット"]
        C["島C：外部知性としてのAIの特性<br/>膨大な知を持つ異物としての刺激<br/>パーソナライズの限界"]
    end
    A <--> B
    B <--> C
```

**KJ法の文章まとめ（Gem出力）：**

> 「私たちがAIと対峙するとき、二つの相反する可能性が共存している。一つは、AIという外部知性を鏡とすることで、一人では到達できなかった自己変容を起こす道。もう一つは、考える主体の維持という境界線を越えてはならないという警告。AIを真に思考を深める道具とするためには、適切な足場かけ設計を自ら行い、AIという異物との摩擦を楽しみながら、自らの主体性を手放さないというマインドセットが不可欠である。」

### 干渉縞 vs KJ法の違い

| | 干渉縞（解説的学習スクリーン） | KJ法 |
|--|------|------|
| 特徴 | 複雑性を保つ。立ち現れないものまで拾う | 収束して綺麗にまとまる |
| 向いている場面 | 多様な声がどう共存しているか観察したいとき | 全体でどういうことなのか理解したいとき |
| 対話への影響 | 摩擦・ねじれを可視化して次の対話へ | まとまった知識として共有・理解 |
| 感触 | 複雑なままで刺激的 | 安心感・わかりやすい |

> 「まとめ方はまさに装置。どの装置を使うかで向かう方向が変わる。何をしたいかによって問いの種類もまとめ方も変わる。」（田原さん）
