## 核心概念①｜反射 vs 解説（カレン・バラードより）

```mermaid
graph LR
    subgraph 反射["【反射 Reflection】"]
        Q1[問い] --> S1[生徒]
        S1 --> A1[一つの答え]
    end
    subgraph 解説["【解説 Diffraction】"]
        Q2[問い] --> SA[生徒A]
        Q2 --> SB[生徒B]
        Q2 --> SC[生徒C]
        SA --> INT[干渉縞✨]
        SB --> INT
        SC --> INT
    end
```

> **反射** = 鏡のように、決まった答えが返ってくる。全員から同じ答えが戻る。
> **解説** = 回析格子のように、多様な答えが広がり、重なり合って新しいパターンが生まれる。

**ポイント：生徒一人ひとりが「スリット（溝）」になる**
光が溝を通って広がるように、問いが生徒を通して多様な回答に広がっていく。

## 核心概念②｜干渉縞の2パターン

```mermaid
graph TD
    A[問いを投げる] --> B[多様な答えが生まれる]
    B --> C{干渉縞}
    C -->|強め合う| D[明線<br/>建設的干渉]
    C -->|弱め合う| E[暗線<br/>相殺的干渉]
    D --> F["異なる視点が補完し合い<br/>一人では到達できない洞察が生まれる"]
    E --> G["矛盾する意見がぶつかり<br/>前提が問い直される"]
    F --> H[コヒーレント<br/>全体の調和・深い納得]
    G --> H
    H --> A
```

### 強め合い（明線）

> 異なる視点が出会うことで、**一人では到達できなかった深い洞察や新しいコンセプト**が立ち上がる。
> お互いの意見の足りないところを補い合って、どちらからとも言えない良い考えが生まれる。

### 弱め合い（暗線）

> 矛盾する意見がぶつかり、**既存の固定観念が揺さぶられ、前提が問い直される**。
> ※悪いことではなく、新しい前提の発見につながる！

**例：数学の背理法**

```
仮定「√2 は有理数（整数/整数で表せる）」
         ↓ どんどん進めると…
              矛盾が生じる！
         ↓
「あの前提がおかしかったんだ」と気づく
         ↓
「無理数」という新しい概念に到達 ✨
```

> 「矛盾が出てくるということは、その矛盾を生み出す前提に何か問題があるんじゃないか。前提を置き直せば、その矛盾は解けるんじゃないか」 — 田原さん

## 核心概念③｜AIをスクリーンとして活用する

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  教師             生徒（スリット）    AI（スクリーン） │
│                                                      │
│  [問いを投げる] ─→ [生徒A] ────────────────┐         │
│                 ─→ [生徒B] ────────────────┼→ [干渉縞│
│                 ─→ [生徒C] ────────────────┤  を可視化│
│                 ─→ [生徒D] ────────────────┘  する]  │
│                                                      │
│  AIが多様な回答を分析・統合し、                      │
│  パターンを「見える化」する                          │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**AIスクリーンの3つの評価軸：**

| 評価軸 | 内容 |
|--------|------|
| 建設的干渉（明線） | 補完し合う意見のクラスタ。強め合っている回答群 |
| 相殺的干渉（暗線） | 矛盾・対立する意見の発見。前提を揺さぶるもの |
| 分布と境界 | まだ観測されていない視点・空白地帯の可視化 |

### ② AIとの実際のやりとり（ステップ詳解）

```mermaid
flowchart TD
    A["Step1: 文脈共有の会話から始める<br/>──────────────────<br/>田原: 「カレン・バラードの解説概念について教えて」<br/>AI: カレンバラードが提唱する解説は、物理学の概念を<br/>社会的な現象・知識の生成プロセスに応用した<br/>非常に独創的でパワフルな概念です…<br/>（反射から解説へ、の考え方を説明）"]
    --> B["Step2: アイデアを投げかける<br/>──────────────────<br/>田原: 「解説を学習に応用したい。学習者集団に<br/>問いを投げると様々な回答が生成される。<br/>これを解説・回析格子と対応付けたいがどうだろう？」<br/>AI: 非常に鋭い着眼点です。学習者集団の<br/>相互作用や問いのプロセスに応用するのは<br/>動的な知識生成を捉えることができます…"]
    --> C["Step3: 設計を依頼する<br/>──────────────────<br/>田原: 「教師が問いを投げ、生徒（スリット）を通して解説し、<br/>干渉縞ができるスクリーンとしてAIを活用します。<br/>どのような設計にすればいいですか？」<br/>AI: 入力=教師の問い＋全生徒の回答テキスト<br/>処理=回答群をベクトル空間に配置し、<br/>各回答の拡散方向を特定。評価軸は3つ…"]
    --> D["Step4: プロンプト生成を依頼<br/>──────────────────<br/>田原: 「AIをスクリーンとして機能させるための<br/>GemのプロンプトをAIに作成させる」<br/>※AIと十分文脈が共有できたタイミングで依頼"]
    --> E["Step5: 細かい調整<br/>──────────────────<br/>AI: 「運用のためには最初にコンテキスト・問い・<br/>回答群の流し込みをやるといいですよ」<br/>田原: 「いや、それもプロンプトに入れて」<br/>→ 3ステップ入力をGemに組み込んでもらう"]
    --> F["✅ Gem完成！（約30分）"]
```

### ③ 大事なポイント

> 「いきなり『作って』ではなく、まずAIと文脈を共有する。
> AIに設計を言わせると、自分が思っているものと違ったりするので、
> やりとりしながら自分が納得するように作っていく。
> それが納得のいくGemへの道。」
